Necesito que desarrolles una herramienta de análisis de sitios web para identificar
prospectos potenciales para servicios de renovación web.

═══════════════════════════════════════════════════════════════════════
CONTEXTO
═══════════════════════════════════════════════════════════════════════

Ofrezco servicios de renovación web para negocios locales pequeños. Necesito una
herramienta CLI que analice una URL y determine automáticamente si el sitio es un
buen candidato para renovación.

═══════════════════════════════════════════════════════════════════════
STACK TECNOLÓGICO
═══════════════════════════════════════════════════════════════════════

- Python 3.10+
- Bibliotecas: requests, beautifulsoup4, playwright (para scraping dinámico)
- CLI con argumentos
- Output en JSON y formato legible en terminal

═══════════════════════════════════════════════════════════════════════
FUNCIONALIDADES REQUERIDAS
═══════════════════════════════════════════════════════════════════════

El script debe analizar una URL y extraer/calcular:

1. DETECCIÓN DE TECNOLOGÍA
   ─────────────────────────────────────────────────────────────────
   □ ¿Es WordPress? (buscar wp-content, wp-includes en HTML)
   □ Si es WP: ¿Qué theme usa?
   □ Si es WP: ¿Usa page builder? (Elementor, WPBakery, Divi, Beaver Builder)
   □ Otras plataformas: Wix, Shopify, Squarespace, HTML estático
   □ Framework JS: React, Vue, Angular (detectar en source)
   □ CMS: Joomla, Drupal

2. ANÁLISIS DE DISEÑO
   ─────────────────────────────────────────────────────────────────
   □ Antigüedad estimada (basado en patrones de diseño):
     - Detectar jQuery UI viejo
     - Detectar Bootstrap 2/3 (vs 4/5)
     - Detectar patrones visuales antiguos (sliders Flash, tables para layout)
   □ Copyright year en footer (buscar © 2015, etc.)
   □ Última actualización de blog/noticias (si existe)

3. ANÁLISIS DE PERFORMANCE
   ─────────────────────────────────────────────────────────────────
   □ Tiempo de carga (primera carga completa)
   □ Tamaño de página (KB/MB)
   □ Número de requests HTTP
   □ Imágenes sin optimizar (detectar imágenes >500KB)
   □ PageSpeed score estimado (o integrar con API de PageSpeed Insights)

4. ANÁLISIS DE RESPONSIVE
   ─────────────────────────────────────────────────────────────────
   □ Meta viewport presente
   □ Media queries detectadas en CSS
   □ Captura screenshot móvil (viewport 375x667)
   □ Detectar elementos que se salen de viewport móvil

5. ANÁLISIS SEO BÁSICO
   ─────────────────────────────────────────────────────────────────
   □ Título de página (length, calidad)
   □ Meta description (presente, length)
   □ H1 tags (cuántos, estructura)
   □ Imágenes sin alt text
   □ URLs amigables vs parámetros (?id=123)
   □ SSL/HTTPS (sí/no)
   □ Sitemap.xml existe

6. ANÁLISIS DE CONTENIDO
   ─────────────────────────────────────────────────────────────────
   □ Número de páginas detectadas (crawl limitado o sitemap)
   □ Número de posts/artículos (si es blog)
   □ Última fecha de actualización visible
   □ Email de contacto visible (extraer)
   □ Teléfono visible (extraer)
   □ Dirección física (extraer)

7. PROBLEMAS TÉCNICOS
   ─────────────────────────────────────────────────────────────────
   □ Enlaces rotos (404)
   □ Imágenes que no cargan
   □ JavaScript errors en console
   □ Certificado SSL inválido
   □ Errores visibles en página

8. ANÁLISIS DE NEGOCIO
   ─────────────────────────────────────────────────────────────────
   □ Tipo de negocio (detectar por keywords: restaurante, peluquería, etc.)
   □ Reseñas de Google integradas (detectar widget)
   □ Formulario de contacto presente
   □ Mapa de Google integrado

═══════════════════════════════════════════════════════════════════════
SISTEMA DE PUNTUACIÓN (0-100)
═══════════════════════════════════════════════════════════════════════

El script debe calcular un score de "potencial de renovación" basado en:

PUNTOS POSITIVOS (indican buen prospecto):
─────────────────────────────────────────────────────────────────────
+15 puntos: Es WordPress sin page builder (escalable)
+10 puntos: Diseño anticuado (>5 años estimado)
+10 puntos: Performance mala (carga >3s)
+10 puntos: No responsive (no mobile-friendly)
+10 puntos: Problemas técnicos visibles (errores, 404s)
+8 puntos:  SEO malo (sin meta tags, sin HTTPS)
+7 puntos:  Contenido desactualizado (último post >1 año)
+5 puntos:  Menos de 20 páginas (manejable)
+5 puntos:  Email/teléfono visible (contactable)
+5 puntos:  Negocio local pequeño (1 ubicación)
+5 puntos:  Reseñas buenas en Google (>4 estrellas)
+5 puntos:  Imágenes sin optimizar
+5 puntos:  Copyright antiguo (© <2020)

PUNTOS NEGATIVOS (indican mal prospecto):
─────────────────────────────────────────────────────────────────────
-20 puntos: Usa page builder pesado (Elementor, Divi)
-15 puntos: Diseño muy moderno (<2 años)
-15 puntos: Sitio muy grande (>50 páginas)
-10 puntos: E-commerce complejo (cientos de productos)
-10 puntos: No es WordPress ni estático (Wix, Squarespace)
-10 puntos: Performance excelente (carga <1s)
-8 puntos:  Es una cadena/franquicia (múltiples ubicaciones)
-5 puntos:  React/Vue/Angular (app moderna)

SCORE FINAL:
─────────────────────────────────────────────────────────────────────
0-30:   ❌ Mal prospecto (rechazar)
31-50:  🟡 Prospecto regular (evaluar caso por caso)
51-70:  ✅ Buen prospecto (contactar)
71-100: 🌟 Prospecto excelente (prioridad máxima)

═══════════════════════════════════════════════════════════════════════
CATEGORIZACIÓN AUTOMÁTICA
═══════════════════════════════════════════════════════════════════════

Además del score, el script debe clasificar el sitio en:

CATEGORÍAS DE ACCIÓN:
─────────────────────────────────────────────────────────────────────
1. "PRIORIDAD_MAXIMA" - Score >70, WordPress simple, <20 páginas
2. "CONTACTAR" - Score 51-70, factible
3. "EVALUAR" - Score 31-50, requiere análisis manual
4. "RECHAZAR" - Score <30, no viable

RAZONES PRINCIPALES:
─────────────────────────────────────────────────────────────────────
Listar las 3 razones principales por las que es/no es buen prospecto

Ejemplos:
- "WordPress sin page builder + diseño anticuado + mal responsive"
- "Usa Elementor (no escalable) + sitio grande (50+ páginas)"

Análisis básico:

python web_analyzer.py https://novanorte.com


Con opciones:
─────────────────────────────────────────────────────────────────────
# Output en JSON
python web_analyzer.py https://novanorte.com --json

# Guardar resultado en archivo
python web_analyzer.py https://novanorte.com --output resultado.json

# Análisis profundo (incluye PageSpeed API)
python web_analyzer.py https://novanorte.com --deep

# Análisis batch (múltiples URLs)
python web_analyzer.py --batch urls.txt

# Solo mostrar score
python web_analyzer.py https://novanorte.com --score-only

REQUERIMIENTOS TÉCNICOS ESPECÍFICOS
═══════════════════════════════════════════════════════════════════════

1. MANEJO DE ERRORES
   ─────────────────────────────────────────────────────────────────
   □ Timeout después de 30 segundos
   □ Si página no carga, devolver partial analysis
   □ Graceful degradation (si un módulo falla, continuar)

2. RATE LIMITING
   ─────────────────────────────────────────────────────────────────
   □ No hacer más de 5 requests/segundo a mismo dominio
   □ Respetar robots.txt (opcional, flag --ignore-robots)

3. USER AGENT
   ─────────────────────────────────────────────────────────────────
   □ Usar User-Agent realista (Chrome/Firefox)
   □ Identificarse: "WebAnalyzer/1.0 (Prospect Validator)"

4. CACHE
   ─────────────────────────────────────────────────────────────────
   □ Opcional: cachear resultados por 24h (flag --cache)
   □ Guardar en .cache/ para no re-analizar mismo sitio

5. PRIVACIDAD
   ─────────────────────────────────────────────────────────────────
   □ No guardar cookies
   □ No ejecutar trackers
   □ No enviar datos a terceros (excepto PageSpeed API si --deep)
